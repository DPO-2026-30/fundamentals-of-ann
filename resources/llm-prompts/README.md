# Промпты для БЯМ и правила использования генеративного ИИ

## Общие правила (из методических указаний и РПД)

- Генеративный ИИ (большие языковые модели — ChatGPT, GigaChat, YandexGPT и др.) допустимо использовать **для пояснений и отладки**.
- Итоговый код и решение должны быть понятны обучающемуся и воспроизведены/объяснены на защите.
- На зачёте использование ИИ и внешних источников запрещено (кроме явно разрешённых).

> Подробные правила для студентов и преподавателей — в [`../../methodical-guidelines/`](../../methodical-guidelines/README.md).

## Анти-чит

- Ключевые алгоритмы (операции с матрицами без numpy, backprop, функция метрик, реализация оптимизаторов) выполняются вручную.
- Метрики и графики строятся по собственным данным (seed по варианту), выводы — по собственным результатам.
- Код и вердикты автотестов — только вход в проверку; главное — защита решения.

## Пример промпта для самопроверки студента

> «Я реализовал [алгоритм X] для задачи [Y]. Объясни мне, как работает шаг [Z], и укажи, где в моём коде могут быть ошибки. Не давай готовое решение — помоги понять».

## Пример промпта для преподавателя (генерация вариантов)

> «Сгенерируй N вариантов параметров для ПР [N] (seed, число признаков, batch_size, lr) в диапазоне [...]. Для каждого варианта укажи ожидаемый качественный результат (переобучение/недообучение/оптимум).»
