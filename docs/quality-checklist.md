Используется перед сдачей ФОС и при последующих изменениях. Все пункты должны быть отмечены перед защитой / публикацией.

## Содержание

- [ ] Корневой [`README.md`](../README.md) содержит название, аннотацию, модель измерения, КИМ, итоговую оценку.
- [ ] РПД ([`docs/rpd.md`](rpd.md)) содержит цель, задачи, пререквизиты, результаты обучения, структуру.
- [ ] Указаны целевые роли КРМ (TADS — якорная; рост к ML Engineer, Data Analyst, ML Researcher) и ожидаемые уровни (MF-1.2 — С; DL-1.x — Б).
- [ ] Коды компетенций и индикаторов сверены с [`data/krm-v3.0.xlsx`](../data/krm-v3.0.xlsx) и [`data/competency-catalog.csv`](../data/competency-catalog.csv).
- [ ] Модель измерения определена для всех модулей (М1, М2) и интегральных КИМ (Project, Exam).
- [ ] Каждый результат обучения подтверждён хотя бы одним КИМ.
- [ ] Каждый КИМ содержит: назначение, проверяемые результаты, условия, материалы, формат сдачи, критерии и шкалу, правила использования ИИ.
- [ ] У каждой рубрики описаны уровни 10 / 8 / 6 / 4 / 0–3.
- [ ] Баллы рубрик согласованы с итоговой таблицей (БРС, 100 баллов).
- [ ] Описаны правила использования генеративного ИИ и внешних ресурсов (анти-чит).
- [ ] Методические указания заполнены (students, teachers-assessment, teachers-resources).
- [ ] Ресурсы сопровождаются карточками (аннотация, связанные КИМ, доступ, лицензия, дата проверки).
- [ ] Заполнены [`team/README.md`](../team/README.md) (авторы, роли, вклад) и [`LICENSE.md`](../LICENSE.md).
