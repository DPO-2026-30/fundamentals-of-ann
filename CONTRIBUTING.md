# Правила внесения изменений

Принцип: **любое изменение должно улучшать проверяемость, прозрачность и воспроизводимость** фонда оценочных средств (ФОС) и его связь с компетентностно-ролевой моделью (КРМ) версии 3.0.

## Процесс

1. Создать ветку от `main`:
   - `feature/<краткое-название>` — для нового содержимого (новый КИМ, рубрика, ресурс);
   - `fix/<краткое-название>` — для исправления ошибки в существующих материалах.
2. Одно логически завершённое изменение — один коммит.
3. Понятное сообщение в формате *Conventional Commits*: тип (`feat`, `fix`, `docs`, `refactor`, `chore`) + краткое описание.
   Пример: `feat: добавить КИМ-05 «Метрики классификации» и рубрику-05`.
4. Перед коммитом убедиться, что:
   - все относительные Markdown-ссылки рабочие;
   - разделы заполнены (нет оставленных `[ЗАПОЛНИТЬ]`);
   - баланс трудоёмкости сходится (Раздел 1: ПЗ = 8 ч; Раздел 2: ПЗ = 22 ч; всего ПЗ = 30 ч);
   - рубрики согласованы с итоговой шкалой оценивания (см. [`README.md`](README.md#итоговая-оценка));
   - привязка к КРМ корректна (код/индикатор/уровень совпадают с [`data/krm-v3.0.xlsx`](data/krm-v3.0.xlsx)).
5. Открыть Pull Request в `main`. В описании указать:
   - какие результаты обучения / компетенции / индикаторы затронуты;
   - какие КИМ изменены или добавлены;
   - что требует обсуждения.

## Требования к КИМ

- **Наблюдаемость:** результат выполнения КИМ наблюдаем и проверяем.
- **Соответствие уровню:** критерии и сложность соответствуют уровню (Б/С/П) по КРМ.
- **Согласованность баллов:** баллы рубрики совпадают с моделью итоговой оценки.
- **Устойчивые ссылки:** внешние материалы указаны по постоянным ссылкам (DOI, официальные репозитории).
- **Анти-чит:** явно описаны правила использования внешних ресурсов и генеративного ИИ (вариантность, защита, формат сдачи).

## Соглашения об именовании

| Объект | Шаблон | Пример |
| --- | --- | --- |
| Папка модуля | `M<номер>-<короткое-название>` | `M1-fundamentals-of-ann` |
| КИМ | `kim-<номер>-<название>.md` | `kim-05-classification-metrics.md` |
| Рубрика | `rubric-<номер>-<название>.md` | `rubric-05-classification-metrics.md` |
| Приложение | `appendix-<номер>-<название>.<расширение>` | `appendix-01-mnist-sample.csv` |

## Ограничения

- **Не редактировать** [`data/krm-v3.0.xlsx`](data/krm-v3.0.xlsx) — это исходная книга КРМ 3.0. Коды, индикаторы и уровни переносятся в документы дисциплины с сохранением формулировок.
- **Не использовать** устаревший репозиторий `aimclub/ai-competency-model` — опираться только на КРМ 3.0.
- **Не размещать** файлы, защищённые авторским правом (полнотекстовые учебники, статьи). В каталоге [`resources/`](resources/README.md) — только библиографические карточки со ссылками.
- **Бинарные файлы больше 10 МБ** в историю Git не помещать — использовать Git LFS или внешние ссылки.
