# КИМ-01. Практическая работа 1 «Основы матричной алгебры. Библиотека numpy»

## 1. Назначение и формируемые результаты

КИМ формирует и проверяет базовый навык реализации операций над матрицами на Python — фундамент для всей последующей работы с ИНС. Результат: обучающийся реализует класс операций с матрицами без готовых функций библиотеки, проверяет корректность через `numpy` и интерпретирует результаты.

## 3. Цель работы

Изучить особенности реализации операций с матрицами на Python.

## 4. Условия проведения

- **Аудитория:** компьютерный класс (корп. 2, ауд. 106 или эквивалент).
- **ПО:** Python 3.10+, numpy. См. [`resources/software/python-libs/`](../resources/software/python-libs/README.md).
- **Время:** 2 ак. часа (1 пара).
- **Вариантность:** seed генератора псевдослучайных чисел определяется номером варианта студента (закрепляется на первом занятии).

## 5. Материалы и ресурсы

- numpy — [`resources/software/python-libs/`](../resources/software/python-libs/README.md).
- Лекция 1 «Основы матричной алгебры на Python».

## 6. Задание

1. Создать класс операций с матрицами, реализующий сложение матриц, умножение матрицы на число, умножение матриц и транспонирование **без использования готовых методов numpy** (через циклы и списки).
2. Проверить работу класса на простых примерах вручную.
3. Создать матрицы размером **не менее 10×10** с помощью `numpy.random.randint()` (seed — по варианту).
4. Применить к созданным матрицам методы своего класса.
5. Сверить результаты методов своего класса с результатами соответствующих методов `numpy`.

## 9. Правила использования внешних ресурсов и генеративного ИИ

- **Разрешено:** пользоваться документацией numpy, справочниками Python.
- **Генеративный ИИ (ChatGPT, GigaChat, YandexGPT и др.):** можно использовать для пояснений и отладки, **но итоговый код должен быть понятен студенту и воспроизведён/объяснён на защите**. Прямое копирование решений без понимания = 0 баллов по критерию «понимание на защите».
- **Анти-чит:** ключевые операции должны быть реализованы вручную (без `numpy.dot`, `numpy.transpose` и т. п.); сверки с `numpy` — обязательны.
